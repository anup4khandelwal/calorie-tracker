# Mise — Architecture

Mise (*mise en place*) is a conversational calorie tracker for iOS. One unified surface:
each **day is a chat thread** with a friendly nutrition agent; **pinch out** of any thread
to see a **magazine timeline** of every meal, rendered as consistent studio-style
generated photography.

## Big picture

```
┌──────────────────────────────────────────────────────────────┐
│ RootView (ZoomStage)                                         │
│  ├── DayPagerView            ← horizontal pager, 1 page/day  │
│  │     └── DayThreadView     ← chat thread for that day      │
│  │           ├── EntryCardView (generated food imagery)      │
│  │           ├── StreamingTextView (custom TextRenderer)     │
│  │           └── ComposerBar                                 │
│  └── TimelineView            ← zoomed-out magazine catalog   │
│        └── MealCard grid, grouped by day                     │
└──────────────────────────────────────────────────────────────┘
        AppModel (@Observable, owns zoom state + services)
        ├── AgentSession (per-day agentic loop)
        │     ├── ClaudeClient       (SSE streaming, tool use)
        │     └── AgentToolbox       (executes tools on SwiftData)
        ├── FoodImageEngine          (Gemini image gen + disk cache)
        ├── NutritionDB              (bundled seed database)
        └── SwiftData: DayLog, ChatMessage, FoodEntry, UserProfile
```

## The three moves

1. **Talk** — the composer sends a user turn into `AgentSession`. The agent streams back
   (Claude `POST /v1/messages`, `stream: true`) and calls tools mid-turn to log/update
   food. Token streaming renders through a custom `TextRenderer` (per-glyph blur+rise
   reveal), so text *condenses* onto the page rather than teletyping.
2. **See** — every logged entry spawns a `FoodEmbed` in the thread: a studio-style
   image generated by Gemini (`gemini-3.1-flash-image`) with a locked style prompt so
   all food photography looks like one magazine shoot. Images develop in with a
   film-grain dissolve shader.
3. **Zoom** — pinch inward on a thread (or tap the date masthead) and the chat recedes
   into a scrollable catalog of meal cards. Pinch/tap a card to dive back into that
   day's conversation, landing scrolled to that entry.

## Agent design

- **Model**: `claude-opus-4-8` via the Messages API (SSE). No SDK — a small hand-rolled
  client (`ClaudeClient`) speaks the wire format directly.
- **System prompt** carries: persona (warm, brief, zero moralizing), today's date, the
  user's goals/profile, today's running totals, and yesterday's summary — so most turns
  need zero tool calls to feel personal.
- **Tools** (executed locally against SwiftData, results returned as `tool_result`):
  | tool | purpose |
  |---|---|
  | `search_food_db` | ground nutrition facts in the bundled DB |
  | `log_food` | create entries (batch), returns ids + updated totals |
  | `update_food_entry` / `delete_food_entry` | edit by id |
  | `get_day` | full log + totals for any date |
  | `query_history` | ranged stats: totals, averages, streaks, top foods |
  | `set_profile` | update goals (calories, protein, name) |
- **History**: during a turn the session keeps exact wire blocks (`tool_use` /
  `tool_result` pairing rules). Persisted history is simplified to text + entry
  references; on reopen, context is rebuilt from the system prompt + text history.

## Imagery pipeline

`FoodImageEngine.image(for: FoodEntry)`:
1. Normalize the food name → cache key (SHA256) → check disk (`Application Support/FoodImages`).
2. Miss → `generateContent` on `gemini-3.1-flash-image` (fallback `gemini-2.5-flash-image`)
   with the locked *Mise shoot* style prompt; decode inline base64 PNG; store.
3. UI shows a shimmering ceramic-plate placeholder (Metal shimmer), then the real image
   arrives via a grain-dissolve `colorEffect`.

## Shaders (`Mise/Shaders/Mise.metal`)

| shader | type | used for |
|---|---|---|
| `plateShimmer` | colorEffect | placeholder while images generate |
| `grainReveal` | layerEffect | film-grain dissolve when an image lands |
| `liquidGlass` | layerEffect | lensing/refraction on the composer + masthead chrome |
| `zoomRipple` | distortionEffect | thread ↔ timeline transition wobble |
| `heatHaze` | layerEffect | subtle idle life on the hero card |

Plus native `MeshGradient` (animated by `TimelineView`) as the ambient background.

## Data model (SwiftData)

- `DayLog(dayKey "yyyy-MM-dd", messages: [ChatMessage], entries: [FoodEntry])`
- `FoodEntry(name, emoji, meal, grams, calories, protein, carbs, fat, createdAt, imagePath?)`
- `ChatMessage(role, text, createdAt, entryIDs)`
- `UserProfile(name, calorieGoal, proteinGoal, apiKeysLiveInKeychain)`

`dayKey` strings keep predicates trivial and timezone-stable.

## Keys & settings

Claude + Gemini API keys are entered in Settings (or first-run) and stored in the
**Keychain** (`KeyVault`). Nothing sensitive is in source or UserDefaults.

## Why no .xcodeproj in the repo

The project is generated with XcodeGen (`project.yml`) to keep the repo reviewable.
`xcodegen generate && open Mise.xcodeproj` is the entire setup.
